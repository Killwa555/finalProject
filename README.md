# 🛡️ AI-Driven Financial Fraud Detection System
**نظام كشف الاحتيال المالي الموجه بالذكاء الاصطناعي**

مشروع تخرج أكاديمي يمثل تقاطعاً متقدماً بين **الأمن السيبراني** و**الذكاء الاصطناعي**. يهدف النظام إلى حماية الشبكات المالية من العمليات الاحتيالية والهجمات السيبرانية المالية من خلال تحليل تدفقات البيانات في الوقت الفعلي (Real-Time) باستخدام خوارزميات تعلم الآلة (Machine Learning)، مع توفير تفسيرات دقيقة لقرارات الذكاء الاصطناعي باستخدام تقنية (Explainable AI - SHAP).

🎓 **جامعة نجران (Najran University)**

---

## 📌 نظرة عامة على المشروع (Project Overview)
يُعد رصد الاحتيال المالي أحد أهم تحديات الأمن السيبراني الحديث. بدلاً من الاعتماد على القواعد التقليدية (Rule-based Systems) التي يسهل اختراقها، يعتمد هذا النظام على نماذج ذكاء اصطناعي تم تدريبها على مجموعة بيانات **IEEE-CIS Fraud Detection** الضخمة لاكتشاف الأنماط الشاذة والمشبوهة. 

لتعزيز موثوقية النظام، تم دمج تقنية **SHAP (SHapley Additive exPlanations)** لتحويل الذكاء الاصطناعي من "صندوق أسود" إلى نظام شفاف قادر على تبرير قراراته للمحلل الأمني (SOC Analyst)، مع توثيق كافة الهجمات/العمليات في قاعدة بيانات آمنة ولامركزية.

---

## 🏗️ المخطط الهندسي للنظام (System Architecture)
يعتمد المشروع على معمارية خدمات متكاملة (Full-Stack Architecture) تضمن السرعة، الأمان، والكفاءة:

1. **الواجهة الأمامية (Frontend - React):**
   * واجهة مراقبة أمنية تفاعلية (Dashboard) مبنية بـ React و Vite.
   * معالجة آمنة لملفات الـ CSV الضخمة وتقسيمها برمجياً في المتصفح.
   * عرض حي (Live Feed) للعمليات مع رسوم بيانية تفسيرية (XAI Modals).

2. **الواجهة الخلفية (Backend - FastAPI):**
   * محرك وسيط (REST API) خفيف وسريع الاستجابة.
   * يعمل كنقطة فحص أمنية تستقبل العمليات، تجهز البيانات، وتمررها لنموذج الذكاء الاصطناعي بأمان.

3. **محرك الذكاء الاصطناعي (AI & ML Engine):**
   * نموذج مصنف (Classifier) مدرب مسبقاً باستخدام `PyCaret`.
   * استخراج مبررات القرار والعوامل التي أدت للاشتباه في المعاملة باستخدام `SHAP TreeExplainer`.

4. **قاعدة البيانات (Database - MongoDB):**
   * استخدام قاعدة بيانات غير علائقية (NoSQL) لتخزين سجلات العمليات بسرعة فائقة.
   * ربط غير متزامن (Asynchronous) عبر مكتبة `Motor` لضمان استمرارية عمل السيرفر أثناء تدفق البيانات.

## 🚀 متطلبات التشغيل (Prerequisites)
تأكد من تثبيت البرامج التالية على جهازك للتشغيل المحلي (Localhost):
* **[Node.js](https://nodejs.org/):** (الإصدار 18 أو أحدث) لتشغيل الواجهة الأمامية.
* **[Python](https://www.python.org/):** (الإصدار 3.9 أو أحدث) لتشغيل الواجهة الخلفية والذكاء الاصطناعي.
* **[MongoDB Community Server](https://www.mongodb.com/try/download/community):** قاعدة البيانات الأساسية للنظام.
* **[MongoDB Compass](https://www.mongodb.com/products/tools/compass):** الواجهة الرسومية لاستعراض قاعدة البيانات المحفوظة.

## ⚙️ طريقة التشغيل (Setup & Installation)

### الخطوة 1: تجهيز قاعدة البيانات
1. تأكد من أن خدمة **MongoDB** تعمل في خلفية نظام التشغيل.
2. افتح برنامج **MongoDB Compass** واضغط على `Connect` باستخدام الرابط الافتراضي: `mongodb://localhost:27017`

### الخطوة 2: تشغيل الواجهة الخلفية (Backend)
افتح الطرفية (Terminal) وانتقل إلى مجلد الباك إند، ثم نفذ الأوامر التالية:
```bash
# تثبيت المكتبات الخاصة بالذكاء الاصطناعي وقواعد البيانات
pip install -r requirements.txt

# تشغيل خادم المراقبة
python main.py


الخطوة 3: تشغيل الواجهة الأمامية (Frontend)
افتح طرفية جديدة وانتقل إلى مجلد الفرونت إند، ثم نفذ:

# تثبيت الحزم الخاصة بـ React
npm install

# تشغيل خادم التطوير
npm run dev
